# Framings

Named structural insights, applied by classification: "does `applies-when` match, and does
`misfires-when` NOT?" Cite by name when used.

---

### overload-behind-conflicts
When several interpersonal problems surface at once around one person, suspect that person's
remaining capacity before treating each conflict individually. Conflicts are often the
symptom, not the cause.
- born: 2026-07, distilled from live sessions
- applies-when: two or more troubles involving the same person, with different counterparties, in a short window
- misfires-when: the counterparties share a structural cause (policy change, reorg) — reducing it to the person misdiagnoses

### shared-cause-behind-parallel-friction
When several *different* people independently raise friction about the same process or area in
a short window, suspect one upstream change they all touched before reading each pushback as
its own interpersonal problem. The common object is the tell.
- born: 2026-07, distilled from a live session
- applies-when: multiple unrelated people push back in the same window, and their complaints point at the same process/system/policy
- misfires-when: the frictions are about genuinely unrelated things with no shared object — then treat each on its own terms

### suspect-the-instrument
A failing test, monitor, or check cannot distinguish "the subject broke" from "the check
broke." Verify the instrument before indicting the subject.
- born: 2026-07, distilled from a live debugging session
- applies-when: the claim "X is broken" rests on a single verification channel's failure
- misfires-when: multiple independent channels show the same failure — instrument-doubt is then a waste of time
