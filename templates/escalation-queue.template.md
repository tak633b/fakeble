# Escalation Queue

Questions that need a stronger pass than the current session can give — banked here instead of answered shallowly. When a stronger model runs (a `/model` switch, a fork, or a fresh frontier session), it works through the open items; every answer that yields a reusable *shape* gets distilled into `framings.md` (see the framing-library skill). This file is a distillation pipeline, not a waiting room.

Each entry is the question distilled to one sentence, just enough context to answer it cold, the date banked, and a checkbox. Check it off once a stronger model has answered it and any framing has been harvested.

---

## Open

- [ ] **<question distilled to one sentence>** — <context hint: enough for a cold read to answer without the original conversation> (banked YYYY-MM-DD)

## Answered

- [x] **<question>** — <context hint> (banked YYYY-MM-DD, answered YYYY-MM-DD) → framing: `<framing-name>` in framings.md, or "no reusable framing"

---

Example open entry:

- [ ] **Resolve the module-ownership overlap between Alex and Kim by splitting the module, or by naming a single owner?** — Both have worked the billing module for a year; Kim asked for ownership in March, Alex just shipped the larger rewrite. Ambiguous value judgment (fairness to Kim's ask vs. momentum behind Alex's work), and it shapes a conversation happening next week. (banked 2026-07-04)
